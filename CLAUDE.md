# CLAUDE.md

Планировщик встреч (тестовое задание). Сотрудник добавляет встречу (кто с кем, когда);
любой другой видит расписание на день/неделю; система не даёт забронировать слот,
если у кого-то из участников уже есть встреча в пересекающееся время.

## Стек

- **FastAPI** — бэкенд: API, бизнес-логика, доступ к БД. Здесь живёт вся логика.
- **Streamlit** — фронтенд: формы и расписание. Тонкий клиент, ходит в API по HTTP.
- **SQLite + SQLAlchemy** — хранилище, ORM.
- **uv** — пакетный менеджер. Используй `uv add` / `uv sync` / `uv run`, **не pip напрямую**.
- **Docker** — контейнеризация. API и UI — отдельные процессы; base URL API у Streamlit через env (`API_BASE_URL`, дефолт `http://localhost:8000`).

## Команды

- `make validate` — линтер + форматтер + тайп-чекер. **Прогоняй перед тем, как считать задачу готовой.**
- `make test` — тесты (pytest). **Прогоняй перед тем, как считать задачу готовой.**
- Makefile уже есть — используй существующие цели, не дублируй и не переопределяй их.

## Доменное правило (критично — ядро задачи)

Запрет пересечений ломать нельзя. Логика:

- Интервалы **полуоткрытые** `[starts_at, ends_at)`. Встречи впритык
  (10:00–11:00 и 11:00–12:00) **НЕ** конфликтуют.
- Два интервала пересекаются ⟺ `s1 < e2 AND s2 < e1`.
- При создании встречи: если хотя бы один участник занят в пересекающемся интервале —
  отказ (HTTP 409) с указанием, **кто именно** и **когда** занят.
- Проверка занятости и вставка — в **одной транзакции** (защита от гонки);
  в SQLite запись сериализуется, этого достаточно.
- Время храним и сравниваем в **UTC**; в UI показываем в локальной TZ.

Тесты на функцию пересечения обязательны — граничные случаи: впритык, вложенный
интервал, частичное перекрытие с обеих сторон, одинаковые границы.

## Модель данных

- `employees(id, name)`
- `meetings(id, title, starts_at, ends_at)` — время в UTC
- `meeting_participants(meeting_id, employee_id)` — M2M, минимум 2 участника

## Соглашения

- Схема создаётся один раз через `Base.metadata.create_all(engine)` на старте
  (идемпотентно). **Alembic не используем** — схема фиксированная, миграции ни к чему.
- Без настоящей авторизации: «кто я» — выпадающий список сотрудников, не логин/пароль.
- Не оверинжинирь. Это тестовое: соразмерность объёму важнее лишнего тюлинга и абстракций.
  Не добавляй SPA, Postgres, Redis, фоновые воркеры и т.п. без явной просьбы.
