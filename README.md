# meeting_calendar

Планировщик встреч. Сотрудник добавляет встречу (кто с кем, когда); любой другой
видит расписание на день/неделю; система не даёт забронировать слот, если у кого-то
из участников уже есть встреча в пересекающееся время.

## Стек

- **FastAPI** — бэкенд (API + вся бизнес-логика).
- **Streamlit** — тонкий фронтенд (формы и расписание, ходит в API по HTTP).
- **SQLite + SQLAlchemy** — хранилище и ORM.
- **uv** — пакетный менеджер. **Docker** — контейнеризация.

## Доменное правило

Интервалы полуоткрытые `[starts_at, ends_at)`: встречи впритык (10:00–11:00 и
11:00–12:00) **не** конфликтуют. Два интервала пересекаются ⟺ `s1 < e2 AND s2 < e1`.
При создании встречи, если хотя бы один участник занят в пересекающемся интервале —
отказ **HTTP 409** с указанием, кто именно и когда занят. Проверка занятости и
вставка выполняются в одной транзакции. Время хранится и сравнивается в UTC, в UI
показывается в локальной таймзоне.

## Структура

```
app/            # FastAPI: config, db, models, schemas, conflicts, crud, routers, main
  routers/      # employees.py, meetings.py
ui/             # Streamlit-клиент (app.py)
tests/          # pytest: логика пересечений + API
```

## Запуск локально

Установка зависимостей:

```bash
uv sync
```

API (порт 8000):

```bash
uv run uvicorn app.main:app --reload
```

Схема БД создаётся на старте, справочник сотрудников засеивается автоматически.
Документация API — http://localhost:8000/docs.

UI (порт 8501, в отдельном терминале):

```bash
uv run streamlit run ui/app.py
```

## Запуск в Docker

Оба процесса через docker-compose:

```bash
docker compose up --build
```

- API — http://localhost:8000
- UI — http://localhost:8501

| `DATABASE_URL`  | API       | `sqlite:///./meeting_calendar.db`   | Строка подключения к БД        |

## API

- `GET /employees` — список сотрудников.
- `POST /meetings` — создать встречу (`title`, `starts_at`, `ends_at`,
  `participant_ids` — минимум 2). `201` при успехе, `409` при конфликте (с указанием
  кто и когда занят), `422` при невалидном вводе.
- `GET /meetings?start=<ISO>&end=<ISO>&employee_id=<опц.>` — расписание за окно
  `[start, end)`; UI формирует окна дня/недели, опциональный фильтр по сотруднику.
