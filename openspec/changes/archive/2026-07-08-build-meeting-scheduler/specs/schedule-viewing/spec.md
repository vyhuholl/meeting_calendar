## ADDED Requirements

### Requirement: View schedule for a day
Система SHALL предоставлять расписание встреч за конкретный день. Возвращаемые
встречи MUST включать те, чей интервал пересекается с выбранным днём. Каждая
встреча в ответе MUST содержать `title`, `starts_at`, `ends_at` и список
участников.

#### Scenario: Встречи за день возвращаются
- **WHEN** клиент запрашивает расписание на конкретный день
- **THEN** система возвращает HTTP 200 и список встреч, пересекающихся с этим днём,
  с их участниками

#### Scenario: День без встреч
- **WHEN** клиент запрашивает расписание на день, в который встреч нет
- **THEN** система возвращает HTTP 200 и пустой список

### Requirement: View schedule for a week
Система SHALL предоставлять расписание встреч за неделю (семидневный интервал).
Возвращаемые встречи MUST включать все встречи, пересекающиеся с этим интервалом.

#### Scenario: Встречи за неделю возвращаются
- **WHEN** клиент запрашивает расписание на неделю
- **THEN** система возвращает HTTP 200 и список встреч, пересекающихся с этой
  неделей

### Requirement: Filter schedule by employee
Система SHALL позволять фильтровать расписание по конкретному сотруднику, возвращая
только встречи, в которых он участвует.

#### Scenario: Фильтр по сотруднику
- **WHEN** клиент запрашивает расписание за период с указанием `employee_id`
- **THEN** система возвращает только те встречи выбранного периода, в которых этот
  сотрудник является участником

#### Scenario: Без фильтра — все встречи
- **WHEN** клиент запрашивает расписание за период без указания сотрудника
- **THEN** система возвращает все встречи периода независимо от участников

### Requirement: Timezone handling in schedule
Система SHALL хранить и сравнивать время в UTC, а UI SHALL показывать время в
локальной таймзоне пользователя.

#### Scenario: Хранение в UTC, отображение локально
- **WHEN** UI отображает встречу, время которой хранится в UTC
- **THEN** UI конвертирует и показывает `starts_at`/`ends_at` в локальной таймзоне
