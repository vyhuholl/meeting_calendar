## ADDED Requirements

### Requirement: Create meeting with participants
Система SHALL позволять создавать встречу с полями `title`, `starts_at`, `ends_at`
и списком участников. Встреча MUST иметь не менее двух участников. Время
`starts_at` и `ends_at` MUST храниться в UTC, и `starts_at` MUST быть строго
раньше `ends_at`.

#### Scenario: Успешное создание встречи
- **WHEN** клиент отправляет `POST /meetings` с валидными `title`, `starts_at`,
  `ends_at` и минимум двумя участниками, ни у одного из которых нет пересекающейся
  встречи
- **THEN** система создаёт встречу, связывает участников и возвращает HTTP 201 с
  созданной встречей (включая её `id` и список участников)

#### Scenario: Меньше двух участников
- **WHEN** клиент отправляет `POST /meetings` с менее чем двумя участниками
- **THEN** система отклоняет запрос с HTTP 422 и не создаёт встречу

#### Scenario: Начало не раньше конца
- **WHEN** клиент отправляет `POST /meetings`, где `starts_at >= ends_at`
- **THEN** система отклоняет запрос с HTTP 422 и не создаёт встречу

#### Scenario: Несуществующий участник
- **WHEN** клиент отправляет `POST /meetings` со ссылкой на `employee_id`,
  которого нет в справочнике
- **THEN** система отклоняет запрос с ошибкой (HTTP 422 или 404) и не создаёт
  встречу

### Requirement: Half-open interval overlap semantics
Система SHALL трактовать интервалы встреч как полуоткрытые `[starts_at, ends_at)`.
Два интервала пересекаются тогда и только тогда, когда `s1 < e2 AND s2 < e1`.
Встречи впритык (конец одной равен началу другой) MUST NOT считаться конфликтом.

#### Scenario: Встречи впритык не конфликтуют
- **WHEN** проверяются интервалы `10:00–11:00` и `11:00–12:00`
- **THEN** функция пересечения возвращает «нет пересечения»

#### Scenario: Одинаковые границы (полное совпадение)
- **WHEN** проверяются интервалы `10:00–11:00` и `10:00–11:00`
- **THEN** функция пересечения возвращает «есть пересечение»

#### Scenario: Вложенный интервал
- **WHEN** один интервал `10:00–12:00` полностью содержит другой `10:30–11:00`
- **THEN** функция пересечения возвращает «есть пересечение»

#### Scenario: Частичное перекрытие слева
- **WHEN** проверяются интервалы `10:00–11:00` и `10:30–11:30`
- **THEN** функция пересечения возвращает «есть пересечение»

#### Scenario: Частичное перекрытие справа
- **WHEN** проверяются интервалы `10:30–11:30` и `10:00–11:00`
- **THEN** функция пересечения возвращает «есть пересечение»

#### Scenario: Непересекающиеся интервалы
- **WHEN** проверяются интервалы `10:00–11:00` и `13:00–14:00`
- **THEN** функция пересечения возвращает «нет пересечения»

### Requirement: Conflict rejection on overlapping booking
Система SHALL отказывать в создании встречи, если хотя бы у одного из участников
уже есть встреча в пересекающемся интервале. Отказ MUST возвращать HTTP 409 и
указывать, **кто именно** из участников занят и **когда** (пересекающийся
интервал или конфликтующая встреча).

#### Scenario: Участник занят в пересекающееся время
- **WHEN** клиент создаёт встречу, где участник уже имеет встречу с пересекающимся
  интервалом
- **THEN** система возвращает HTTP 409, тело ответа называет занятого сотрудника и
  время конфликта, и новая встреча не создаётся

#### Scenario: Впритык — конфликта нет
- **WHEN** клиент создаёт встречу `11:00–12:00`, а у участника уже есть встреча
  `10:00–11:00`
- **THEN** система создаёт встречу и возвращает HTTP 201

#### Scenario: Конфликт хотя бы по одному участнику
- **WHEN** для одного участника конфликта нет, но для другого участника есть
  пересекающаяся встреча
- **THEN** система возвращает HTTP 409 и не создаёт встречу

### Requirement: Atomic conflict check and insert
Система SHALL выполнять проверку занятости участников и вставку встречи в **одной
транзакции**, чтобы исключить гонку между параллельными бронированиями. В SQLite
запись сериализуется, и этого достаточно.

#### Scenario: Проверка и вставка атомарны
- **WHEN** создаётся встреча
- **THEN** проверка пересечений участников и запись встречи с участниками
  происходят в рамках одной транзакции; при обнаружении конфликта транзакция не
  оставляет частично записанных данных
